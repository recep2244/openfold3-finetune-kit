01 — The gap

Co-folding models memorise. They don't extrapolate.

On the Runs N' Poses benchmark — 2,600 protein–ligand systems released after the models' training cutoff — leading co-folding methods were shown to largely reproduce poses seen in training. Accuracy falls as chemistry drifts from what the model has seen.

The implication: a general checkpoint is often weakest exactly where drug discovery lives — novel chemical matter on your target.

02 — The approach

A gentle nudge, not a retrain.

Targeted

Fine-tune on ~10 of your own protein–ligand complexes. Interface-biased crops focus learning where binding happens.

Conservative

Low learning rate, short warmup, faster EMA, few steps — adapt to the target without forgetting general structure.

Verifiable

Score baseline vs fine-tuned on a held-out set. Interface lDDT, DockQ and lDDT-PLI up; ligand RMSD down; global lDDT held.

03 — The pipeline

One command, four stages.

  1. 01

    Prepare

    Download structures & fetch MSAs from the ColabFold server. No local databases.

  2. 02

    Preflight

    Validate every artifact before spending GPU time.

  3. 03

    Fine-tune

    Load of3-p2-155k directly; train the low-N recipe.

  4. 04

    Evaluate

    Baseline vs fine-tuned on held-out structures, scored with OpenStructure.

05 — Get going

Edit two lists. Run one command.

bash

# 1 · install OpenFold3 (pixi) + this kit
git clone https://github.com/aqlaboratory/openfold-3.git ~/openfold-3
( cd ~/openfold-3 && pixi run -e openfold3-cuda12 setup_openfold )
git clone https://github.com/recep2244/openfold3-finetune-kit.git
cd openfold3-finetune-kit

# 2 · verify, then run the whole pipeline
make verify
$ bash scripts/run_all.sh
✓ baseline vs fine-tuned scored — results.csv written
